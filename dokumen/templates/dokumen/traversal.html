{% if mode == 'inorder' %}

In-Order mengunjungi node dengan urutan: Kiri dulu, lalu Root, lalu Kanan. Karena BST menyimpan kode lebih kecil di kiri, hasil In-Order selalu terurut naik — inilah mengapa dashboard utama menggunakan traversal ini.

Pola: f(kiri) → kunjungi(root) → f(kanan) ✓ Hasil Terurut
{% elif mode == 'preorder' %}

Pre-Order mengunjungi Root terlebih dahulu, baru subtree Kiri, lalu Kanan. Traversal ini berguna untuk menyalin/merekonstruksi struktur BST, karena root selalu diproses sebelum anak-anaknya.

Pola: kunjungi(root) → f(kiri) → f(kanan) Root → Anak
{% else %}

Post-Order mengunjungi semua anak dulu, baru Root paling akhir. Traversal ini berguna untuk menghapus seluruh tree secara aman — anak dihapus sebelum induknya.

Pola: f(kiri) → f(kanan) → kunjungi(root) Anak → Root
{% endif %}
